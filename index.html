Graph algorithms

Two common representations of graphs: adjacency lists (sparse graphs) and adjacency matrices (dense graphs)

Adjacency lists节省memory但是弊端是判断一个edge \((u,v)\) is present or not只能通过search \(v\) in the adjacency list \(Adj[u]\)，adjacnecy matrices相反

1. Breadth-First Search (BFS)

BFS systematically explores the edges of \(G\) to discover every vertex that is reachable from \(s\). It computes the distance (smallest number of edges) from \(s\) to each reachable vertex.

Breadth-first search is so named because it expands the frontier between discovered and undiscovered vertices uniformly across the breadth of the frontier. That is, the algorithm discovers all vertices at distance \(k\) from \(s\) before discovering any vertices at distance \(k+1\).

BFS colors each vertex white (undiscovered), gray (discovered but children have not been discovered) or black (children have been discovered).

\(u\) is the predecessor or parent of \(v\) in the breadth-fisrt tree if \((u,v)\in E\).

Below shows the BFS algorithm which uses a first-in, first-out queue \(Q\) to manage the set of gray vertices.

  • Running time \(O(V+E)\), which means BFS runs in time linear in the size of the adjacency-list representation of \(G\)
  • Shortest-path distance \(\delta(s,v)\): the minimum number of edges in any path from \(s\) to \(v\); if there is no path from \(s\) to \(v\), then \(\delta(s,v)=\infty\).
  • Lemma: for any edge \((u,v)\in E\), \(\delta(s,v)\leq\delta(s,u)+1\)
  • Lemma: the queue \(Q\) contains vertices \(v_1,\ldots,v_r\) where \(v_1\) is the head of \(Q\) and \(v_r\) is the tail. Then \(v_r.d\leq v_1.d+1\) and \(v_i.d\leq v_{i+1}.d\) for \(i=1,2,\ldots,r-1\).
  • Breadth-first trees: the tree corresponds to the \(\pi\) attributes. Predecessor subgraph of \(G\) is defined as \(G_\pi=(V_\pi,E_\pi)\) where \(V_\pi=\{v\in V:v.\pi\neq NIL\}\cup \{s\}\) and \(E_\pi=\{(v.\pi,v):v\in V_\pi-\{s\}\}\). \(G_\pi\) is a breadth-first tree if \(V_\pi\) consists of the vertices reachable from \(s\) and for all \(v\in V_\pi\), the subgraph \(G_\pi\) contains a unique simple path from \(s\) to \(v\) that is also a shortest path from \(s\) to \(v\) in \(G\).

2. Depth-First Search

Predecessor subgraph of a DFS is slightly different from that of a BFS. \(G_\pi=(V,E_\pi)\), where \(E_\pi=\{(v.\pi,v):v\in V \text{ and } v.\pi\neq NIL\}\). \(G_\pi\) of a DFS forms a depth-first forest comprising several depth-first trees.

The procedure DFS below records when it discovers vertex \(u\) in the attribute \(u.d\) and when if finishes vertex \(u\) in the attribute \(u.f\). For every vertex \(u\), \(u.d<u.f\) and \(u\) is WHILE before time \(u.d\), GRAY between time \(u.d\) and time \(u.f\), and BLACK thereafter.

  • Parenthesis theorem: In any depth-first search of a graph \(G=(V,E)\), for any two vertices \(u\) and \(v\), eaxactly one of the following three conditions holds:
    • \([u.d,u.f]\) and \([v.d,v.f]\) are entirely disjoint, and neither \(u\) nor \(v\) is a descendant of the other in the DFS
    • \([u.d,u.f]\) is contained within \([v.d,v.f]\), and \(u\) is a descendant of \(v\) in DFS
    • \([v.d,v.f]\) is contianed within \([u.d,u.f]\), and \(v\) is a descendant of \(u\) in DFS
  • Nesting of descendants's intervals: \(v\) is a proper descendant of \(u\) in DFS for a graph \(G\) if and only if \(u.d<v.d<v.f<u.f\).
  • White-path theorem: In DFS, \(v\) is a descendant of \(u\) if and only if at the time \(u.d\) that the search discovers \(u\), there is path from \(u\) to \(v\) consisting entirely of white vertices.

Classification of edges: four edge types in the depth-first forest \(G_\pi\) produced by a DFS on \(G\)

  • Tree edges: edges in \(G_\pi\)
  • Back edges: those edges \((u,v)\) connecting a vertex \(u\) to an ancestor \(v\) in a depth-first tree.
  • Forward edges: those nontree edges \((u,v)\) connecting a vertex \(u\) to a descendant \(v\) in a depth-first tree.
  • Cross edges: all other edges (e.g., 连接无ancestor关系的vertices或者不同depth-first trees中的)

When we first explore an edge \((u,v)\), the color of vertex \(v\) determines the type of the edge: WHITE indicates a tree edge, GRAY indicates a back edge and BLACK indicates a forward or cross edge.

In DFS of an undirected graph \(G\), every edge of \(G\) is either a tree edge or a black edge.

3. Applications of DFS

1. Topological sort

Topological sort of a dag \(G=(V,E)\) is a linear ordering of all its vertices such that if \(G\) contains an edge \((u,v)\), then \(u\) appears before \(v\) in the ordering.

Many applications use DAG to indicate precedences among events, like the ordering of wearing clothes.

  • \(O(V+E)\)
  • Lemma: a directed graph \(G\) is acyclic if and only if a depth-first search of \(G\) yields no back edges

2. Strongly connected components

Decomposition of a graph.

A strongly connected component of a directed graph \(G=(V,E)\) is a maximal set of vertices \(C\subseteq V\) such that for every pair of vertices \(u\) and \(v\) in \(C\), we have both \(u\leadsto v\) and \(v\leadsto u\); that is , vertices \(u\) and \(v\) are reachable from each other.

Transpose of \(G\) is defined as \(G^T=(V,E^T)\), where \(E^T=\{(u,v):(v,u)\in E\}\). Given an adjacency-list representation of \(G\), the time to create \(G^T\) is \(O(V+E)\). And \(G\) and \(G^T\) have exactly the same strongly connected components.

Maximum flow

1. Maximum Flow

  • Flow network: directed graph \(G=(V,E)\) where edges have nonnegative capacity \(c(u,v)\) and there is no edge in the reverse direction
  • Source \(s\) and sink \(t\), and each vertex lies on some path from \(s\) to \(t\), so \(|E|>|V|-1\)
  • A flow \(f\) in G statisfies
    • Capacity constraint: \(0\leq f(u,v)\leq c(u,v)\)
    • Flow conservation: for all \(u\in V-\{s,t\}\), \(\sum_{v\in V}f(v,u)=\sum_{v\in V}f(u,v)\), flows in equals flows out
  • The value of a flow \(f\): \(|f|=\sum_{v\in V}f(s,v)=\sum_{v\in V}f(v,s)\), the total flow out of the source minus the flow into the source
  • Maximum-flow problem: wish to find a flow of maximum value

2. The Ford-Fulkerson Method

  • Residual network \(G_f=(V,E_f)\) with \(E_f=\{(u,v)\in V\times V:c_f(u,v)>0\}\)

    • Residual capacity (easy to find \(|E_f|\leq 2|E|\))

      • \[ c_f(u,v)=\bigg\{ \begin{align*} &c(u,v)-f(u,v)&&\text{if }(u,v)\in E,\\ &f(v,u)&&\text{if} (v,u)\in E,\\ &0&&\text{otherwise.} \end{align*} \]

  • Augmentation of flow \(f\) by \(f'\):

    • \[ (f\uparrow f')(u,v)=\bigg\{ \begin{align*} &f(u,v)+f'(u,v)-f'(v,u)&&\text{if }(u,v)\in E,\\ &0&&\text{otherwise.} \end{align*} \]

  • Lemma: a flow network \(G\) with flow \(f\) and a residual network \(G_f\) induced by \(f\) with flow \(f'\), then \(f\uparrow f'\) is a flow in \(G\) with value \(|f\uparrow f'|=|f|+|f'|\)

  • Augmenting path: a simple path \(p\) from \(s\) to \(t\) in the residual network

    • Residual capacity of \(p\): \(c_f(p)=min\{c_f(u,v):(u,v)\text{ is on }p\}\)

    • \[ f_p(u,v)=\bigg\{ \begin{align*} &c_f(p)&&\text{if }(u,v) \text{ is on }p,\\ &0&&\text{otherwise.} \end{align*} \]

      Then \(f_p\) is a flow on \(G\) with value \(|f_p|=c_f(p)>0\)

    • Corollary: \(f\uparrow f_p\) is flow in G with value \(|f\uparrow f_p|=|f|+|f_p|\)

1. Matrix Multiplication Idea

Negative edges my be present but we assume there are no negative-weight cycles.

  • Idea: consider a shortest path \(p\) from vertex \(i\) to \(j\) and suppose that \(p\) contains at most \(m\) edges (因为无negative-weight cycle所以\(m\)有限). If \(i=j\), \(p\) has weight 0; otherwise we decompose path \(p\) into \(i\overset{p'}{\leadsto}k\rightarrow j\), where \(p'\) contains at most \(m-1\) edges, so \(\delta(i,j)=\delta(i,k)+w_{kj}\).

A recursive solution to the all-pairs shortest-paths problem

Let \(l_{ij}^{(m)}\) be the minimum weight of any path from vertex \(i\) to \(j\) that contains at most \(m\) edges. Thus, \[ l_{ij}^{(0)}=\bigg\{ \begin{align*} &0&& \text{if }i=j,\\ &\infty&&\text{if }i\neq j. \end{align*} \] For \(m\geq 1\), \[ \begin{align*} l_{ij}^{(m)}&=&&\text{min}(l_{ij}^{(m-1)}, \underset{1\leq k \leq n, k\neq j}{\text{min}}\{l_{ij}^{(m-1)}+w_{kj}\})\\ &=&&\underset{1\leq k \leq n}{\text{min}}\{l_{ij}^{(m-1)}+w_{kj}\}. \end{align*} \] Since the shortest path from vertex \(i\) to \(j\) contain at most \(n-1\) edges, \(\delta(i,j)=l_{ij}^{(n-1)}=l_{ij}^{(n)}=l_{ij}^{n+1}=\cdots\). So \(L^{n-1}\) is what we want.

Single-source shortest path

1. Shortest-Path Problem

A weighted, directed graph \(G=(V,E)\) with weight function \(w: E\rightarrow\mathbb{R}\).

The weight \(w(p)\) of path \(p=<v_0,v_1,\ldots,v_k>\) is \(w(p)=\sum_{i=1}^kw(v_{i-1},v_i)\) and the shortest-paht weight \(\delta(u,v)\)from \(u\) to \(v\) is \[ \delta(u,v)=\bigg\{ \begin{align*} &\text{min}\{w(p):u\overset{p}{\leadsto}v\} && \text{if there is a path from $u$ to $v$,}\\ &\infty && \text{otherwise.} \end{align*} \] A shortest path from vertex \(u\) to vertex \(v\) is then defined as any path \(p\) with weight \(w(p)=\delta(u,v)\).

Single-source shortest-paths problem: given a graph \(G=(V,E)\), we want to find a shortest path from a given source vertex \(s\in V\) to each vertex \(v\in V\).

  • Lemma: subpaths of shortest paths are shortest paths

A shortest-paths tree rooted at \(s\) is a directed subgraph \(G'=(V',E')\), where \(V'\subseteq V\) and \(E'\subseteq E\), such that

  1. \(V'\) is the set of vertices reachable from \(s\) in G
  2. \(G'\) forms a rooted tree with \(s\), and
  3. for all \(v\in V'\), the unique simple path from \(s\) to \(v\) in \(G'\) in a shortest path from \(s\) to \(v\) in \(G\).
  • Remark
    • If the graph contains a negative-weight cycle reachable from the source \(s\), shortest-path weights are not well defined.
    • WLOG, we assume no cycles in shortest paths, i.e., shortest paths are simple paths.
    • Shortest paths are not necessarily unique and neither are shortest-paths tree.

Dynamic programming

1. Elements of Dynamic Programming

When does the dynamic programming method apply?

  • Two key ingredients that an optimization problems must have in order for dynamic programming to apply: optimal substructure and overlapping subproblems.

1. Optimal Substructure

A problem exhibits optimal substruture if an optimal solution to the problem contains within it optimal solutions to subproblems. (Optimal solutions to a problem incorporate optimal solutions to related subproblems, which we may solve independently)

Optimal substructure varies across problem domains in two ways:

  1. how many subproblmes an optimal solution to the original problem uses

  2. how many choices we have in determining which subproblems to use in an optimal solution

Actually, the running time of a dynamic programming algorithm depends on the product of the two factors above: the number of subproblems overall and how many choives we look at for each subproblem.

2. Overlapping Subproblems

We say that the optimazation problem has overlapping subproblems when a recursive algorithm revisits the same problem repeatedly.

In contrast, a problem for which a divide-and-conquer approach is suitable usually generated brand-new problems at each step of the recursion.

What's the real need for an end-symbol in n-gram models?

We need the end-symbol to make the bigram grammar a true probability distribution. Without an end-symbol, the sentence probabilities for all sentences of a given length would sum to one, and the probability of the whole language would be infinite.

That is, the probabilities of one-word sequences sum to one \(\sum_iP(w_i)=1\), the probabilities of two-word sequences sum to one \(\sum_i\sum_jP(w_iw_j)=1\), \(\ldots\), etc.

Without an end-symbol, there's no distinction between "the probability of seeing two words next to each other" and "the probability of seeing a two-word sentence".

So we add an end-symbol to the bigram language model (n-gram language model: current word depends on previous n-1 words only) to assign a single probability distribution across all sentence lengths, which is \[ P(\mathbf{w})=\prod_{i=1}^{k+1}P(w_i|w_{i-1}) \] where \(w_0\) is a start-symbol and \(w_{k+1}\) is an end-symbol.
